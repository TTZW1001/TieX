## 2. 产品概述

### 2.1 产品背景

现有大模型聊天产品通常更偏向问答。用户获得模型回复后，仍需要手动复制内容、创建文件、整理文档、制作演示文稿或修改代码。

TieX 参考 Codex 桌面端的任务式交互与工作区设计，面向个人使用场景，提供一个可连接自定义大模型 API 的本地桌面 AI 智能体工作台。

用户可以通过自然语言描述任务，由 TieX 调用受控工具，在用户授权的本地工作区中读取文件、生成文档、创建 PPT、修改代码，并展示任务过程、操作结果和文件变更。

### 2.2 产品定位

TieX 的产品定位为：

> 一个可连接自定义大模型 API、支持本地工作区、受限工具调用和人工审批的个人桌面 AI 智能体助手。

TieX V1.0 不追求完整复刻 Codex，也不追求“万能电脑助手”。第一版重点是完成稳定、清晰、可控的单智能体任务执行流程。

### 2.3 产品命名

TieX 由 “Tie” 与 “X” 组成：

- Tie 来源于“小贴”；
- X 代表扩展、执行、未知能力与可持续拓展；
- 产品名称简短，适合桌面应用标题、图标和 GitHub 仓库命名。

## 3. 产品目标

### 3.1 核心目标

1. 支持用户配置并调用 DeepSeek API。
2. 支持普通 AI 聊天和多轮上下文。
3. 支持用户选择本地文件夹作为工作区。
4. 支持智能体在受限工作区中读取、创建和修改文件。
5. 支持生成 Markdown、DOCX 和 PPTX 文件。
6. 支持基础代码分析、修改和受限命令执行。
7. 所有高风险操作必须经过权限判断或用户确认。
8. 所有业务数据默认保存在本地。
9. 用户可随时停止任务、查看修改并恢复文件。

### 3.2 V1.0 非目标

TieX V1.0 暂不实现：

- 用户注册与登录
- 云端同步
- 团队协作
- 插件市场
- MCP 插件系统
- 自动化定时任务
- 多智能体并行
- 远程执行环境
- GitHub 云端任务
- 移动端
- 浏览器自动操作
- 屏幕识别
- 鼠标与键盘自动控制
- 完整虚拟机级沙箱
- 完整终端模拟器
- 任意系统权限操作

## 4. 设计原则

### 4.1 本地优先

- 会话、设置、任务记录和工作区信息默认保存在本地。
- 不建设 TieX 自有云端服务。
- 不要求用户注册账号。
- 不主动上传完整工作区。

### 4.2 用户可控

- 用户明确选择工作区。
- 用户明确选择权限模式。
- 高风险操作需要人工确认。
- 用户可随时终止任务。
- 文件修改必须可查看、可撤销。

### 4.3 最小权限

- 智能体默认只能访问当前授权工作区。
- 工具通过白名单注册。
- 渲染进程不能直接访问文件系统或终端。
- API Key 不以明文保存。

### 4.4 渐进实现

开发顺序遵循：

1. 普通聊天
2. 工作区读取
3. 工具调用
4. 文件修改
5. 文档与 PPT
6. 受限编程执行

## 5. 目标用户与使用场景

### 5.1 目标用户

TieX V1.0 主要供开发者本人使用，不针对特定行业或特定人群进行商业化定位。

### 5.2 普通对话

用户未选择工作区时，可直接与模型对话。

示例：

- 帮我分析一个项目创意。
- 帮我整理一份开发计划。
- 解释这段代码的含义。
- 帮我生成一份文档大纲。

此模式下，TieX 不主动读取本地文件，也不执行本地命令。

### 5.3 文档生成

用户描述文档需求，TieX 生成结构化内容并导出。

示例：

- 生成一份项目需求分析文档。
- 根据当前项目生成 README。
- 将 Markdown 转换成 Word。
- 把材料整理成正式报告。

### 5.4 PPT 生成

用户输入演示主题或添加参考材料，TieX 生成 PPTX 文件。

示例：

- 根据项目文档生成十页答辩 PPT。
- 根据指定主题生成汇报演示。
- 将现有 Markdown 整理成演示文稿。

### 5.5 编程辅助

用户选择代码项目作为工作区，TieX 可分析或修改代码。

示例：

- 阅读项目并说明目录结构。
- 修复登录页面报错。
- 为接口添加参数校验。
- 运行测试并根据结果修改代码。

### 5.6 文件整理

TieX 可在授权工作区中执行受控文件操作。

示例：

- 按文件类型整理资料。
- 批量重命名指定文件。
- 将生成物保存到 `docs` 目录。

批量移动、覆盖、重命名和删除操作必须展示清单并等待确认。

## 10. 工作区需求

### 10.1 选择工作区

用户通过系统文件夹选择器选择一个本地目录。

保存信息：

- 工作区名称
- 本地路径
- 创建时间
- 最近访问时间
- 默认权限模式

### 10.2 工作区访问范围

智能体仅允许访问：

- 当前工作区根目录
- 当前工作区子目录
- TieX 临时目录
- TieX 生成物目录
- TieX 备份目录

默认禁止访问：

- 工作区外目录
- 用户其他个人文件
- 浏览器数据
- 系统目录
- 其他磁盘
- 网络共享路径
- 敏感环境变量

### 10.3 路径安全

所有工具调用必须：

1. 将相对路径解析为绝对路径；
2. 校验目标路径位于当前工作区内；
3. 阻止符号链接绕过；
4. 阻止绝对路径绕过；
5. 阻止 UNC 网络路径；
6. 阻止系统保留路径；
7. 拒绝包含非法路径片段的请求。

### 10.4 工作区切换

切换工作区后：

- 当前会话更新工作区关联；
- 工具权限重新计算；
- 未完成任务必须停止或请求确认；
- 不自动把旧工作区内容带入新任务。

## 11. 沙箱与权限设计

### 11.1 沙箱定义

TieX V1.0 使用“工作区受限沙箱”，不是完整虚拟机沙箱。

工作区受限沙箱包括：

- 文件访问范围限制
- 工具白名单
- 路径校验
- 操作审批
- 命令限制
- 执行超时
- 子进程终止
- 操作日志
- 文件备份
- 修改恢复

### 11.2 权限模式

#### 仅对话模式

允许：

- 与模型聊天
- 读取用户主动添加的附件
- 生成文本回复

禁止：

- 浏览工作区
- 修改文件
- 运行命令

#### 读取模式

允许：

- 查看工作区目录
- 读取工作区文本文件
- 搜索文件名或文本内容
- 分析项目

禁止：

- 创建文件
- 修改文件
- 删除文件
- 运行命令

#### 执行模式

允许：

- 读取文件
- 创建文件
- 修改文件
- 生成文档
- 生成 PPT
- 执行受限命令

以下操作仍必须确认：

- 覆盖已有文件
- 删除文件
- 批量移动或重命名
- 执行终端命令
- 安装依赖
- 联网请求
- 修改项目配置
- 一次修改多个文件
- 访问工作区外路径

### 11.3 操作审批

审批弹窗必须显示：

- 准备执行的操作
- 操作原因
- 目标文件或命令
- 可能影响
- 允许一次
- 本次任务内允许
- 拒绝
- 停止整个任务

V1.0 不提供“永久允许所有危险操作”的默认入口。

### 11.4 命令执行限制

V1.0 不直接向模型开放任意 Shell。

受限命令执行器要求：

- 工作目录固定为当前工作区
- 命令执行前确认
- 命令超时
- 最大输出长度
- 支持终止子进程
- 禁止管理员权限
- 禁止长期交互式进程
- 过滤敏感环境变量
- 检测危险命令
- 记录完整日志

默认禁止：

- 删除磁盘或目录树
- 修改注册表
- 格式化磁盘
- 关机或重启
- 用户账户管理
- 防火墙修改
- 下载并执行未知程序
- 编码或混淆的 PowerShell 命令
- 工作区外文件操作

### 11.5 文件备份与恢复

修改已有文件前自动保存原始版本。

备份记录包括：

- 文件路径
- 原始内容
- 修改时间
- 所属任务
- 修改摘要

用户可：

- 查看修改前后差异
- 恢复单个文件
- 撤销当前任务全部修改

## 12. DeepSeek API 接入需求

### 12.1 基础配置

默认服务商：

```text
服务商：DeepSeek
Base URL：由用户配置或使用默认值
模型名称：由用户配置
API Key：由用户填写
```

模型名称不得硬编码。

### 12.2 API Key 存储

API Key 不得：

- 明文写入项目文件
- 明文存入日志
- 显示完整内容
- 发送给非用户配置服务商

应使用操作系统安全存储或 Electron `safeStorage` 加密保存。

界面仅显示掩码：

```text
sk-••••••••••••9H2K
```

### 12.3 模型能力要求

V1.0 需要支持：

- 多轮对话
- 流式输出
- Tool Calls
- JSON 结构化输出
- 上下文管理
- 请求取消
- 基础错误重试

### 12.4 工具调用流程

```text
用户提交任务
  ↓
系统构建上下文
  ↓
调用 DeepSeek API
  ↓
模型返回文本或工具调用
  ↓
系统校验工具名称和参数
  ↓
系统进行权限判断
  ↓
执行工具
  ↓
将工具结果返回模型
  ↓
继续执行或输出最终结果
```

### 12.5 任务限制

每个任务必须限制：

- 最大模型调用轮数
- 最大工具调用次数
- 最大连续失败次数
- 最大执行时间
- 最大读取文件数
- 最大单文件大小
- 最大上下文长度

用户可随时点击“停止”终止任务。

## 13. Agent Runtime 需求

### 13.1 核心职责

Agent Runtime 负责：

- 接收用户任务
- 构建上下文
- 调用模型
- 解析工具调用
- 校验参数
- 判断权限
- 执行工具
- 回传结果
- 控制任务循环
- 处理中止与错误
- 保存任务日志

### 13.2 任务状态

任务状态包括：

- 等待中
- 思考中
- 等待授权
- 执行工具中
- 生成文件中
- 已完成
- 已停止
- 执行失败

### 13.3 任务停止

用户点击停止后：

- 取消正在进行的模型请求；
- 停止当前工具；
- 终止相关子进程；
- 保存当前任务状态；
- 不再执行后续步骤。

应用关闭时，正在执行的任务必须安全终止，不在后台静默继续。

## 14. 内置工具需求

### 14.1 V1.0 工具列表

| 工具名称 | 功能 | 最低权限 |
|---|---|---|
| `list_files` | 查看工作区目录 | 读取模式 |
| `read_file` | 读取文本文件 | 读取模式 |
| `search_files` | 搜索文件名或内容 | 读取模式 |
| `create_file` | 创建新文件 | 执行模式 |
| `edit_file` | 修改已有文件 | 执行模式 |
| `create_markdown` | 创建 Markdown 文档 | 执行模式 |
| `create_docx` | 创建 Word 文档 | 执行模式 |
| `create_pptx` | 创建 PowerPoint 文件 | 执行模式 |
| `run_command` | 执行受限命令 | 执行模式且需确认 |

### 14.2 工具参数校验

模型返回的工具参数必须经过 Schema 校验。

校验失败时：

- 不执行工具；
- 返回明确错误；
- 允许模型修正参数；
- 连续失败达到上限后停止任务。

### 14.3 工具调用日志

记录：

- 工具名称
- 输入参数
- 权限判断
- 用户审批结果
- 开始时间
- 结束时间
- 执行结果
- 错误信息

日志必须过滤 API Key 和其他敏感值。

## 15. 文档生成功能

### 15.1 支持格式

V1.0 支持：

- Markdown
- DOCX

### 15.2 文档生成流程

1. 用户描述文档需求；
2. 模型生成结构化大纲；
3. 用户确认或直接继续；
4. 模型生成章节内容；
5. 系统调用文档生成工具；
6. 文件保存到工作区或生成物目录；
7. 页面展示文件卡片。

### 15.3 DOCX 基础样式

支持：

- 文档标题
- 一级至三级标题
- 正文段落
- 有序列表

## 21. Electron 安全要求

渲染进程必须设置：

```text
contextIsolation = true
nodeIntegration = false
sandbox = true
```

渲染进程不得直接：

- 访问文件系统
- 执行命令
- 读取 API Key
- 操作 SQLite
- 调用系统级 API

所有本地能力通过 Preload 和受限 IPC 接口提供。

每个 IPC 接口必须：

- 验证参数类型
- 验证路径范围
- 验证调用权限
- 捕获异常
- 记录安全日志

需要注意：

> Electron 的 `sandbox = true` 仅用于渲染进程隔离，不能替代 TieX 的工作区受限沙箱、工具权限和命令执行限制。

## 26. 产品成功标准

TieX V1.0 成功不以功能数量判断，而以以下结果判断：

1. 用户能够稳定连接 DeepSeek API；
2. 普通聊天和历史会话正常使用；
3. 智能体只能访问用户授权工作区；
4. 智能体能完成至少一种真实多步骤任务；
5. 文件修改可查看、可确认、可撤销；
6. 能生成可用的 Markdown、DOCX 和 PPTX 文件；
7. 应用不会在用户不知情时执行危险操作；
8. 模型输出错误时，系统不会直接把错误参数当作可信指令执行。
