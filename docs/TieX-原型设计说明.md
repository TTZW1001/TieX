## 2. 原型目标

本原型用于明确 TieX V1.0 的页面结构、核心交互、视觉方向和主要状态。

原型不承担真实业务功能，不调用 DeepSeek API，不访问本地文件，也不执行任何命令。所有数据和交互均为演示内容。

原型重点验证：

1. 可收起的左侧侧边栏；
2. 首页任务输入区；
3. 普通对话与智能体任务执行界面；
4. 工具调用过程展示；
5. 权限审批弹窗；
6. 文件修改 Diff 展示；
7. 右侧任务详情抽屉；
8. DeepSeek API 设置页；
9. 浅色与深色主题；
10. Codex 风格但不完全照搬的桌面工作台布局。

---

## 3. 设计原则

### 3.1 简洁优先

TieX 是个人使用工具，V1.0 不放置插件、自动化、多智能体、团队协作和云端同步入口。

左侧栏只保留：

- 新对话
- 当前工作区
- 最近会话
- 设置

### 3.2 任务导向

首页不设计传统“功能首页”，而是直接提供任务输入框。

用户打开 TieX 后，可以直接输入：

- 分析项目
- 生成文档
- 制作 PPT
- 修改代码
- 整理文件

### 3.3 安全可见

智能体调用本地工具时，界面必须清楚展示：

- 正在执行什么
- 读取了哪些文件
- 准备修改哪些内容
- 是否需要用户授权
- 修改后如何撤销

### 3.4 弱化后台系统感

TieX 不采用典型后台管理系统样式。

视觉特征：

- 大面积留白
- 柔和边框
- 较大圆角
- 卡片式任务信息
- 低对比度图标
- 少量强调色
- 重点突出对话和任务本身

## 4. 页面结构

TieX V1.0 包含三个主要页面：

1. 首页 / 新对话页
2. 会话与任务页
3. 设置页

此外包含：

- 权限审批弹窗
- 右侧任务详情抽屉
- Toast 状态提示
- 浅色 / 深色主题

---

## 5. 全局布局

整体采用三段式结构：

```text
┌──────────────┬─────────────────────────────────────────┐
│ 左侧侧边栏    │ 顶部标题栏                               │
│              ├─────────────────────────────────────────┤
│ 新对话        │                                         │
│ 当前工作区    │ 主内容区                                 │
│ 最近会话      │                                         │
│              │                                         │
│ 设置          │                                         │
└──────────────┴─────────────────────────────────────────┘
```

### 5.1 左侧侧边栏

展开宽度建议：260～280 px。

收起宽度建议：64～72 px。

展开状态显示：

- TieX Logo
- TieX 名称
- 收起按钮
- 新对话
- 当前工作区
- 最近会话
- 设置

收起状态仅保留图标。

侧边栏状态需要保存在本地。

### 5.2 顶部标题栏

包含：

- 返回
- 前进
- 当前会话标题
- 当前模型
- 主题切换
- 任务详情按钮

Electron 正式实现时可将顶部栏与自定义窗口标题栏合并。

### 5.3 主内容区

根据页面切换为：

- 首页输入区域
- 会话消息列表
- 设置表单

## 6. 首页设计

### 6.1 页面目标

让用户进入应用后立即开始任务，不增加多余导航。

### 6.2 核心内容

页面中央显示：

> 我们来完成什么？

下方显示任务输入框。

输入框底部包含：

- 添加文件
- 权限模式
- 模型
- 当前工作区
- 开始任务

### 6.3 快捷任务卡片

V1.0 可提供三个快捷入口：

- 分析项目
- 生成文档
- 制作 PPT

点击卡片后，只将示例任务填入输入框，不直接执行。

### 6.4 默认状态

- 默认模型：DeepSeek
- 默认权限：读取模式
- 未选择工作区时：仅对话模式
- 已选择工作区时：显示工作区名称

---

## 7. 会话与任务页设计

### 7.1 消息结构

消息分为：

- 用户消息
- TieX 普通回复
- 工具调用卡片
- 文件修改 Diff
- 权限请求
- 文件生成结果
- 错误消息

### 7.2 工具调用卡片

工具调用卡片展示：

- 工具名称
- 当前状态
- 已读取或操作的对象
- 错误信息
- 展开 / 收起

示例：

```text
读取工作区文件                  已完成
--------------------------------------
permission.ts
read-file.ts
path-safe.ts
```

### 7.3 文件修改 Diff

代码修改后必须展示变更摘要。

Diff 至少包括：

- 文件路径
- 删除内容
- 新增内容
- 查看完整差异
- 撤销修改

### 7.4 输入区

会话底部输入区包含：

- 补充任务文本
- 添加附件
- 权限模式
- 当前模型
- 发送按钮
- 执行时的停止按钮

---

## 8. 右侧任务详情抽屉

### 8.1 开启方式

点击顶部“任务详情”按钮打开。

默认关闭。

### 8.2 抽屉内容

建议提供三个标签：

1. 步骤
2. 文件
3. 日志

### 8.3 步骤标签

展示当前任务执行流程：

- 已完成步骤
- 当前步骤
- 等待授权步骤
- 失败步骤

### 8.4 文件标签

展示：

- 已读取文件
- 已创建文件
- 已修改文件
- 生成物

### 8.5 日志标签

展示简化执行日志，不展示模型内部完整推理。

---

## 9. 权限审批弹窗

### 9.1 触发场景

以下操作必须弹出审批：

- 覆盖已有文件
- 删除文件
- 批量移动或重命名
- 执行命令
- 安装依赖
- 修改项目配置
- 访问工作区外路径
- 一次修改多个文件

### 9.2 弹窗内容

审批弹窗必须显示：

- 操作类型
- 操作原因
- 目标文件或命令
- 影响范围
- 是否会自动备份

### 9.3 操作按钮

提供：

- 拒绝
- 本次任务内允许
- 允许一次

V1.0 不提供“永久允许所有危险操作”。

## 10. 设置页设计

设置页分为四个区块。

### 10.1 DeepSeek API

包含：

- 服务商
- 模型名称
- Base URL
- API Key
- 测试连接
- 保存配置

API Key 使用密码输入框显示。

### 10.2 任务与权限

包含：

- 修改文件前确认
- 执行命令前确认
- 修改前自动备份
- 是否允许访问工作区外路径

工作区外访问默认关闭。

### 10.3 外观

包含：

- 跟随系统
- 浅色
- 深色
- 默认侧边栏状态

### 10.4 本地数据

包含：

- 数据保存目录
- 打开目录
- 清理缓存
- 清除全部数据

## 11. 原型交互说明

HTML 原型已实现以下演示交互：

1. 点击左上角按钮收起或展开侧边栏；
2. 点击“新对话”返回首页；
3. 点击最近会话进入会话页；
4. 点击设置进入设置页；
5. 点击快捷任务卡片填充输入框；
6. 点击“开始任务”进入任务页面；
7. 点击顶部任务详情按钮打开或关闭右侧抽屉；
8. 点击主题按钮切换浅色 / 深色主题；
9. 点击“请求应用修改”打开权限审批弹窗；
10. 点击审批按钮显示操作结果提示；
11. 点击“测试连接”显示模拟连接成功提示。

## 12. 视觉规范建议

### 12.1 色彩

建议使用：

- 主背景：浅灰白 / 深灰黑
- 卡片背景：纯白 / 深灰
- 主强调色：蓝紫色
- 成功：绿色
- 警告：橙色
- 错误：红色

### 12.2 圆角

建议：

- 主输入框：20～24 px
- 卡片：14～18 px
- 按钮：10～12 px
- 小标签：999 px

### 12.3 阴影

只在以下组件使用阴影：

- 首页主输入框
- 权限审批弹窗
- 右侧抽屉
- 浮层提示

普通卡片主要使用边框区分，避免界面过重。

### 12.4 字体

Windows 端建议字体顺序：

```css
font-family:
  Inter,
  "Segoe UI",
  "PingFang SC",
  "Microsoft YaHei",
  sans-serif;
```

## 13. 响应式与窗口要求

建议最小窗口尺寸：

- 宽度：1024 px
- 高度：720 px

推荐默认窗口尺寸：

- 宽度：1440 px
- 高度：900 px

当窗口宽度不足时：

- 侧边栏允许自动收起；
- 右侧抽屉以浮层形式覆盖；
- 首页快捷卡片改为单列；
- 设置表单改为单列。

## 15. V1.0 不进入原型的内容

暂不设计：

- 插件页面
- 自动化页面
- 智能体管理页面
- 多智能体并行视图
- 云端任务
- 团队协作
- 登录注册
- 会员与付费
- 插件市场
- 浏览器控制
- 系统级电脑控制

---

## 16. 原型验收标准

原型满足以下条件即可进入开发：

1. 左侧栏可展开和收起；
2. 首页可以直接开始任务；
3. 会话页能够展示普通消息和工具调用；
4. 高风险操作有独立审批流程；
5. 文件修改有 Diff 展示；
6. 右侧抽屉能展示任务步骤；
7. 设置页包含 DeepSeek API 配置；
8. 深浅色主题可切换；
9. 页面风格统一；
10. 原型不存在超出 PRD 的新增功能。

---

## 17. 文件说明

- `TieX-Prototype.html`：可直接用浏览器打开的交互原型。
- `TieX-原型设计说明.md`：供 Trae、Codex 或开发者参考的原型说明文档。

HTML 原型仅用于界面和交互参考，正式开发时应拆分为 Vue 组件，不应将单文件原型直接作为生产代码继续堆叠。
