# 七、数据库要求

## 7.1 数据库位置

```text
%APPDATA%/TieX/tiex.db
```

数据库文件必须放在用户数据目录，不得放在项目安装目录中。

## 7.2 数据库初始化

首次启动时：

1. 检测 `%APPDATA%/TieX/` 目录是否存在，不存在则创建；
2. 创建 `tiex.db` 文件；
3. 启用 WAL 模式：`PRAGMA journal_mode=WAL;`
4. 启用外键：`PRAGMA foreign_keys=ON;`
5. 执行迁移脚本；
6. 插入默认设置数据；
7. 插入默认 DeepSeek 配置。

重复启动时：

1. 连接已有数据库；
2. 启用 WAL 和外键；
3. 检查并执行未执行的迁移；
4. 不重复插入默认数据。

## 7.3 迁移机制

必须实现迁移表 `schema_migrations`：

```sql
CREATE TABLE schema_migrations (
  version TEXT PRIMARY KEY,
  applied_at TEXT NOT NULL
);
```

迁移文件放在：

```text
electron/main/database/migrations/
```

迁移执行逻辑：

1. 读取 `schema_migrations` 中已执行的版本；
2. 对比 `migrations/` 目录中的迁移文件；
3. 按版本号顺序执行未执行的迁移；
4. 每个迁移在事务中执行；
5. 执行成功后写入 `schema_migrations`。

## 7.4 初始表结构

阶段二必须创建以下表（参照 `TieX-数据库设计.md`）：

- `app_settings`
- `model_providers`
- `workspaces`
- `conversations`
- `messages`
- `tasks`
- `task_steps`
- `tool_calls`
- `permission_requests`
- `file_changes`
- `artifacts`
- `operation_logs`
- `schema_migrations`

建表 SQL 必须严格参照 `TieX-数据库设计.md` 中的定义。

- `setSidebarCollapsed(collapsed: boolean): Promise<void>`

## 9.2 API Key 安全存储

- 使用 Electron `safeStorage.encryptString()` 加密 API Key；
- 使用 `safeStorage.decryptString()` 解密；
- 加密后的数据以 BLOB 存入 `model_providers.encrypted_api_key`；
- 不得以明文形式保存 API Key；
- 不得将 API Key 写入日志或控制台。
