# 七、生成物管理要求

## 7.1 Artifact 统一模型

```ts
interface Artifact {
  id: string
  taskId: string
  type: 'markdown' | 'docx' | 'pptx'
  name: string
  relativePath: string
  absolutePath: string
  size: number
  createdAt: string
}
```

## 7.2 生成位置

默认生成到：

```text
工作区/.tiex/artifacts/
```

如果用户未选择工作区，则保存到：

```text
%APPDATA%/TieX/artifacts/
```

生成目录不存在时自动创建。

## 7.3 文件名规则

- 文件名必须非空；
- 不得包含非法字符（`\0`、路径分隔符等）；
- 扩展名必须与工具类型匹配（`.md`、`.docx`、`.pptx`）；
- 文件名非法时返回错误，并告知用户原因；
- 已存在文件进入权限审批流程（复用阶段六）。

## 7.4 ArtifactService 接口

```ts
interface ArtifactService {
  /**
   * 记录生成物
   */
  recordArtifact(input: CreateArtifactInput): ArtifactEntity

  /**
   * 获取任务的生成物列表
   */
  getByTaskId(taskId: string): ArtifactEntity[]

  /**
   * 获取生成物详情
   */
  getById(artifactId: string): ArtifactEntity | null

  /**
   * 打开生成文件（使用 shell.openPath）
   */
  openFile(artifactId: string): Promise<boolean>

  /**
   * 打开所在目录（使用 shell.showItemInFolder）
   */
  openFolder(artifactId: string): Promise<void>

  /**
   * 删除生成物记录（逻辑删除）
   */
  delete(artifactId: string): void
}

interface CreateArtifactInput {
  taskId: string
  workspaceId?: string | null
  artifactType: 'markdown' | 'docx' | 'pptx'
  name: string
  relativePath?: string | null
  absolutePath: string
  mimeType?: string | null
  sizeBytes: number
  fileHash?: string | null
}
```

## 7.5 事件

新增事件类型：

```ts
{ type: 'artifact:created', taskId, artifactId, artifactType, name }
```

# 十、create_pptx 工具要求

## 10.1 用途

根据结构化规格生成 PPTX 演示文稿。

## 10.2 注册名称

```text
create_pptx
```

## 10.3 输入

```ts
interface CreatePptxInput {
  path: string                   // 工作区内的相对路径
  presentation: PresentationSpec // 演示结构化规格
  overwrite?: boolean            // 是否允许覆盖，默认 false
}
```

## 10.4 PresentationSpec

```ts
interface PresentationSpec {
  title: string                  // 演示标题
  subtitle?: string              // 副标题（可选）
  slides: Array<{
    layout: 'cover' | 'agenda' | 'title_content' | 'two_column' | 'image_text' | 'table' | 'summary'
    title: string                // 页面标题
    bullets?: string[]           // 项目符号（适用 title_content、two_column、summary）
    leftContent?: string[]       // 左栏内容（适用 two_column）
    rightContent?: string[]      // 右栏内容（适用 two_column）
    table?: {
      headers: string[]
      rows: string[][]
    }                            // 表格（适用 table）
  }>
}
```

## 10.5 版式说明

| 版式 | 用途 | 元素 |
|------|------|------|
| `cover` | 封面页 | 标题、副标题 |
| `agenda` | 目录/议程页 | 标题、项目符号列表 |
| `title_content` | 标题 + 正文 | 标题、项目符号列表或段落 |
| `two_column` | 双栏布局 | 标题、左右两栏内容 |
| `image_text` | 图文页（预留图片占位） | 标题、正文（V1.0 暂不嵌入图片） |
| `table` | 表格页 | 标题、表格 |
| `summary` | 总结页 | 标题、总结要点列表 |

## 10.6 输出

```ts
interface CreatePptxOutput {
  path: string
  sizeBytes: number
  created: boolean
  artifactId: string
  slideCount: number
}
```

## 10.7 规则

- 后缀必须为 `.pptx`；
- `path` 必须经过 PathGuard 校验；
- 模型不直接生成 XML，仅输出 `PresentationSpec` 结构化数据；
- 使用 `pptxgenjs` npm 包生成 PPTX 文件；
- 使用预设版式，不允许模型传递任意坐标；
- 每页内容居中，不出现文本框溢出；
- 标题字号统一（建议 28-36pt），正文字号统一（建议 18-24pt）；
- 使用简洁配色（深蓝/白/灰配色方案）；
- 限制总页数最多 20 页；
- 每页 bullet 最多 6 项；
- 每项 bullet 最多 40 个字符（防止溢出）；
- 表格页最多 6 列、10 行；
- 默认 `overwrite = false`；
- 写入后记录 artifact（类型 pptx）。

          "type": "array",
          "items": {
            "type": "object",

                "type": "string",

```sql
-- 006_create_artifacts_table.sql

CREATE TABLE artifacts (
  id TEXT PRIMARY KEY,
  task_id TEXT NOT NULL,
  workspace_id TEXT,
  artifact_type TEXT NOT NULL CHECK(artifact_type IN ('markdown', 'docx', 'pptx')),
  name TEXT NOT NULL,
  relative_path TEXT,
  absolute_path TEXT NOT NULL,
  mime_type TEXT,
  size_bytes INTEGER NOT NULL DEFAULT 0,
  file_hash TEXT,
  status TEXT NOT NULL DEFAULT 'created' CHECK(status IN ('created', 'missing', 'deleted')),
  created_at TEXT NOT NULL,
  deleted_at TEXT,
  FOREIGN KEY (task_id) REFERENCES tasks(id) ON DELETE CASCADE,
  FOREIGN KEY (workspace_id) REFERENCES workspaces(id)
);

CREATE INDEX idx_artifacts_task_id ON artifacts(task_id);
CREATE INDEX idx_artifacts_created_at ON artifacts(created_at DESC);
```

# 二十、代码质量要求

1. 使用 TypeScript，避免滥用 `any`；
2. `docx` 和 `pptxgenjs` 的 API 调用需封装，不直接暴露在工具层；
3. 文件写入使用临时文件 + 原子替换（与阶段六一致）；
4. 生成物路径通过 PathGuard 校验；
5. DOCX 和 PPTX 生成内容限制防止内存溢出；
6. PPT 每页 bullet 数量限制防止溢出；
7. 文件名后缀校验在前端和后端均执行；
8. 不破坏阶段一至阶段六已有的界面功能；
9. 不修改文档内容；
10. 不修改原型文件。
